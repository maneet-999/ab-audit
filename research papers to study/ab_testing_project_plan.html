01 — Project Objectives

What We're Building and Why

This project builds a statistical validity audit engine that diagnoses A/B tests for methodological failures before teams make incorrect shipping decisions. The primary goal is to produce a portfolio artifact that demonstrates graduate-level statistical reasoning, not just code.

OBJ-01 — Primary

Build a working diagnostic tool

A deployed Streamlit app that audits A/B experiment setups and results across 8 statistical validity checks, returning plain-English verdicts with quantified error consequences.

OBJ-02 — Primary

Produce original empirical research

Run 1,000 synthetic experiments with known ground truth. Quantify the percentage that would be incorrectly shipped without the audit. This number is the paper's headline finding.

OBJ-03 — Secondary

Write a research-style paper

A full write-up with abstract, methodology, results, and citations — formatted like a conference paper. Publishable on arXiv as a technical report.

OBJ-04 — Secondary

Create an interview-ready demo

A scripted 10-minute demo walkthrough of the tool using the Zepto scarcity badge scenario. Every module explained, every violation surfaced, verdict shown live.

04 — Statistical Theory Used

The Academic Foundation

Every module maps directly to formal statistical theory. This section is your study guide — the concepts you must deeply understand before building each check.

Hypothesis Testing Core

Null / Alternative Hypotheses

H₀: p₁ = p₂ (no effect). H₁: p₁ ≠ p₂ (effect exists). Two-tailed tests throughout. Understanding directional vs. non-directional hypotheses. Rejection region derivation.

Error Types

Type I & Type II Errors

α = P(reject H₀ | H₀ true) = false positive rate. β = P(fail to reject H₀ | H₁ true) = false negative rate. Power = 1 − β. The core tradeoff the entire tool is built around.

Sampling Theory

Central Limit Theorem

Why sample proportions are approximately normal for large n. The foundation for z-tests on proportions. When CLT breaks down (small n, extreme proportions) and why you need Fisher's exact test.

Power Analysis

Sample Size Determination

Deriving n = 2(z_α/2 + z_β)² p(1−p) / δ² from first principles. Minimum detectable effect. How power collapses as baseline rate approaches 0 or 1. Effect size (Cohen's h).

Multiple Testing

Family-Wise Error Rate

FWER = 1−(1−α)^k inflation. Bonferroni correction: αᵢ = α/k. Benjamini-Hochberg procedure for False Discovery Rate control. When to use which correction and why.

Sequential Testing

Optional Stopping Problem

Why checking p < 0.05 repeatedly inflates Type I error. Sequential Probability Ratio Test (SPRT). Always-valid p-values (Johari et al. 2015). Spending functions for error control.

Nonparametric

Distribution-Free Tests

When metric distributions violate normality. Shapiro-Wilk test. Mann-Whitney U as t-test alternative. Levene's test for variance homogeneity. Welch's t-test for unequal variances.

Variance Reduction

CUPED Method

Controlled-experiment Using Pre-Experiment Data. Covariate adjustment: Ỹ = Y − θX where θ = Cov(Y,X)/Var(X). Reduces variance of treatment effect estimator without bias. Equivalence to OLS residuals.

Goodness of Fit

Sample Ratio Mismatch

Chi-square test on observed vs. expected assignment counts. H₀: randomization worked. Detecting pipeline bugs before they corrupt results. Chi-square distribution properties.

Causal Inference

SUTVA & Network Effects

Stable Unit Treatment Value Assumption. When social features violate independence. Cluster randomization as the correct design. Interference detection heuristics.

Bayesian Methods

Beta-Binomial Conjugate Model

Prior: Beta(α, β). Posterior: Beta(α + successes, β + failures). Credible intervals vs. confidence intervals. Probability that treatment beats control: P(p₂ > p₁). Why Bayesian allows peeking without the same penalty.

Simulation Theory

Monte Carlo Methods

Law of Large Numbers justifying empirical estimates. Generating Bernoulli and Normal random variables. Variance of Monte Carlo estimator. Why 10,000 simulations gives stable estimates.

Theoretical References — What to Read

Sequential

Johari, Pekelis & Walsh (2015) — Always Valid Inference

The peeking problem paper. Derives always-valid p-values using mixture sequential probability ratio tests. Core theory for Module 6.

CUPED

Deng, Xu, Kohavi & Walker (2013) — Improving Sensitivity via Pre-Experiment Data

The original CUPED paper from Microsoft Research. Derive the covariate adjustment estimator and its variance reduction properties.

Industry

Kohavi, Tang & Xu (2020) — Trustworthy Online Controlled Experiments

The definitive industry textbook. Chapters on SRM, multiple testing, novelty effects, and experiment design. Read chapters 3, 5, 8, and 19.

Classical

Fisher (1935) — The Design of Experiments

Foundational. Read the first three chapters. Citing Fisher signals you understand the historical foundation, not just the applied version.

Multiple

Benjamini & Hochberg (1995) — Controlling the False Discovery Rate

The BH procedure paper. Read the original — it's short (12 pages). Understand the step-up procedure. Required for Module 5.

08 — Risks & Mitigation

What Can Go Wrong and How to Handle It

Scope creep — building too many features
High risk

Mitigation

Lock scope at end of Phase 0. Any new feature proposed after Day 3 goes into a "backlog" file — not the build. The 8 checks are fixed. Nothing gets added until all 8 are done and deployed.

Statistical implementation bugs
High risk

Mitigation

Every check is unit tested against SciPy's equivalent function with identical inputs. If our output differs by more than 1e-6, there is a bug. Never trust your implementation without this cross-validation step.

Monte Carlo too slow for UI
Med risk

Mitigation

Use vectorized NumPy from day one — no Python loops in the simulation. Add st.cache_data decorator on heavy computation. If still slow, add a progress bar and async loading. Target: 10,000 sims in under 5 seconds.

CUPED derivation is incorrect
Med risk

Mitigation

Validate CUPED output against a manual OLS regression on the same data — they should produce identical adjusted estimates. Cross-check variance reduction ratio with the Deng 2013 formula. If they match, the implementation is correct.

Paper not written — only the app ships
Med risk

Mitigation

Write the paper abstract and methodology section in Week 2, not Week 5. The paper is much easier to write once the simulation results exist. Set a 2-hour block every day in Week 5 exclusively for paper writing.

Deployment issues on Streamlit Cloud
Low risk

Mitigation

Test deployment on Day 22 — not Day 28. Leave 3 days buffer for debugging dependency issues. Pin all library versions in requirements.txt. Have a Hugging Face Spaces backup deployment ready if Streamlit fails.

No users — nobody discovers the tool
Low risk

Mitigation

On deployment day: post on LinkedIn with a clear problem statement ("Most A/B tests are statistically invalid — here's a tool to catch it before you ship"). Tag 3–5 data science communities. Post in r/datascience. The topic is inherently shareable.

def check_srm(n_control, n_treatment, expected_ratio=0.5) -> CheckResult: n_total = n_control + n_treatment expected_control = n_total * expected_ratio expected_treatment = n_total * (1 - expected_ratio) chi2 = ((n_control - expected_control)**2 / expected_control + (n_treatment - expected_treatment)**2 / expected_treatment) p_val = 1 - scipy.stats.chi2.cdf(chi2, df=1) passed = p_val > 0.01 return CheckResult( name="Sample Ratio Mismatch", passed=passed, severity="pass" if passed else "fail", statistic=round(chi2, 4), p_value=round(p_val, 4), verdict="Randomization looks clean." if passed else f"SRM detected (p={p_val:.4f}). Assignment split is {n_control/n_total:.1%}/{n_treatment/n_total:.1%} instead of 50/50.", cost_of_violation="Your confidence intervals are unreliable.
